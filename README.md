# Proteome Profiler Array Analysis

This repository analyzes membrane-based Proteome Profiler arrays from LI-COR spot-intensity exports.

It supports one pipeline with two analysis modes:

- legacy exploratory mode: fold-change visualization only
- replicate-aware mode: multiplicity-corrected inferential testing

All analyses are configured through `input$manifest`. The manifest is the source of truth for workbook locations and experimental design.

The protocol-to-analyte workbook generated by the Python setup step is required in both cases.

If you want rendered output examples, see [docs/README.md](docs/README.md).

## Quick Start

1. Install R packages:

```bash
Rscript scripts/install_packages.R
```

2. Generate the protocol workbook:

```bash
python3 scripts/setup/extract_analyte_table.py --preset cytokine_xl
```

3. Edit [scripts/config/analysis_config.R](scripts/config/analysis_config.R):
   - set `runtime_root`
   - set `cloud_parent`
   - add or edit one entry under `proteome_profiler_config$analyses`

4. Choose the analysis to run:

```bash
export PROTEOME_PROFILER_ANALYSIS=your_analysis_name
```

5. Review and choose the low-signal threshold inputs:

```bash
Rscript scripts/find_ref_thresh.R
```

This is a manual review checkpoint, not just another command in a chain.

After `find_ref_thresh.R` finishes, you must:

- look at your data and the suggested low-signal analytes
- decide which coordinates belong in `thresholds$ref_coords`
- re-run `Rscript scripts/find_ref_thresh.R` if you changed that coordinate panel
- read `threshold_diagnostics/region_stats.png`
- choose `thresholds$ref_signal` from that diagnostic before running `main.R`

Refer to [docs/README.md](docs/README.md) for more details on interpreting the threshold diagnostics.

6. Run the main analysis:

```bash
Rscript scripts/main.R
Rscript scripts/select-analytes-analysis.R
```

`find_ref_thresh.R` is optional in replicate-aware runs, but if you want a low-signal heuristic at all, this manual interpretation step still happens before `main.R`.

## How To Run Your Own Analysis

You do not need to copy the VEGFRi/Dox structure.

To run your own project, you edit one analysis entry in [scripts/config/analysis_config.R](scripts/config/analysis_config.R) and point it at your own files. The shipped VEGFRi/Dox entry is only a worked reference.

The active analysis is selected by `PROTEOME_PROFILER_ANALYSIS`. The shared config does not silently choose one for you.

### Minimal Legacy Analysis Entry

Use this shape when you want exploratory outputs only.

```r
my_project = list(
  user = "nick",
  slug = "my_project_slug",
  mode = "legacy",
  protocol = list(
    preset = "cytokine_xl",
    workbook = "output/cytoXL array kit - protocol.xlsx"
  ),
  input = list(
    manifest = "manifests/my_project_samples.csv",
    treatment = "treatment"
  ),
  comparisons = list("vehicle" = c("treated")),
  thresholds = list(
    ref_coords = c("A3,4"),
    ref_signal = c(150),
    fold_change = c(1.5),
    groups_per_page = 25
  ),
  shortlist = list(
    control = "vehicle",
    treatment = "treated",
    fold_change = 1.5
  )
)
```

### Minimal Replicate-Aware Analysis Entry

Use this shape when you have one workbook per biological sample and want inferential testing:

```r
my_project = list(
  user = "lauren",
  slug = "aldo_plasma_cytokine_xl",
  mode = "replicate",
  protocol = list(
    preset = "cytokine_xl",
    workbook = "output/cytoXL array kit - protocol.xlsx"
  ),
  input = list(
    manifest = "manifests/aldo_samples.csv",
    subgroup = "sex",
    treatment = "treatment"
  ),
  comparisons = list("vehicle" = c("aldosterone")),
  thresholds = list(
    ref_coords = c("A3,4"),
    ref_signal = c(150)
  ),
  stats = list(
    min_reps_per_arm = 2,
    p_adjust_method = "BH",
    alpha = 0.05
  ),
  shortlist = list(
    comparisons = c("male_vehicle_vs_aldosterone", "female_vehicle_vs_aldosterone"),
    top_n = 10
  )
)
```

## Workflow

### Step 1: Build The Protocol Workbook

Run:

```bash
python3 scripts/setup/extract_analyte_table.py --preset cytokine_xl
```

This creates the workbook that maps array coordinates to analyte names. The R scripts depend on that mapping. Without it, the spot labels are not reliable.

### Step 2: Define The Analysis In `analysis_config.R`

Create or edit one named entry under `proteome_profiler_config$analyses`.

Required top-level fields:

- `user`
- `slug`
- `protocol`
- `comparisons`

Common grouped fields:

- `protocol$preset`
- `protocol$workbook`
- `thresholds$ref_coords`
- `thresholds$ref_signal`

Preferred shared input fields:

- `input$manifest`
- `input$treatment`

Exploratory-only fields:

- `thresholds$fold_change`
- `thresholds$groups_per_page`
- `shortlist$control`
- `shortlist$treatment`
- `shortlist$fold_change`

Replicate-aware input fields:

- `input$subgroup`
- `stats$min_reps_per_arm`
- `stats$p_adjust_method`
- `stats$alpha`
- `shortlist$comparisons`
- `shortlist$top_n`

### Step 3: Put The Input Files In Place

For both modes:

- put one LI-COR workbook per sample in whatever location you want
- define sample identity and design in a manifest

Required manifest columns:

- `sample_id`
- `workbook_path`
- `treatment`

Add any subgroup column you want to stratify by, for example `sex`.

Example manifest:

```csv
sample_id,workbook_path,treatment,sex
M01,data/m01.xlsx,vehicle,male
M02,data/m02.xlsx,vehicle,male
M03,data/m03.xlsx,aldosterone,male
M04,data/m04.xlsx,aldosterone,male
F01,data/f01.xlsx,vehicle,female
F02,data/f02.xlsx,vehicle,female
F03,data/f03.xlsx,aldosterone,female
F04,data/f04.xlsx,aldosterone,female
```

In both modes, filenames should not be treated as the source of truth. The manifest is.

### Step 4: Run `find_ref_thresh.R` And Interpret It Before `main.R`

Run:

```bash
Rscript scripts/find_ref_thresh.R
```

This writes:

- `threshold_diagnostics/candidate_low_signal_analytes.tsv`
- `threshold_diagnostics/region_stats.png`

This is the manual decision step between config setup and `main.R`.

Do not treat this as "run one script and immediately run the next one." The point of `find_ref_thresh.R` is to help you choose two config values:

- `thresholds$ref_coords`
- `thresholds$ref_signal`

What you do with it:

1. Review your data and inspect `threshold_diagnostics/candidate_low_signal_analytes.tsv`.
2. Decide which analytes are genuinely sitting near the boundary between usable and unusable signal for your dataset.
3. Put those coordinates into `thresholds$ref_coords`.
4. Re-run `Rscript scripts/find_ref_thresh.R` if you changed that panel.
5. Open `threshold_diagnostics/region_stats.png`.
6. Use the mean signal and magenta reference line to choose `thresholds$ref_signal`.
7. Write that value into [scripts/config/analysis_config.R](scripts/config/analysis_config.R).
8. Only then run `Rscript scripts/main.R`.

Example of the diagnostic you are interpreting:

![Threshold diagnostics example](docs/output-examples/threshold-diagnostics-region-stats.png)

What the example is showing:

- the top panel shows the candidate low-signal analytes across groups
- the lower-left histogram shows the signal distribution with the chosen mean threshold marked in magenta
- this figure is for illustration only; you still need to choose coordinates and a threshold based on your own dataset

Why the coordinate choice is manual:

- the script can suggest low-signal analytes
- it cannot know which weak analytes make assay-level sense to use as the low-signal reference panel for your experiment
- that is why the coordinate-selection step exists at all

In short:

- first choose `thresholds$ref_coords`
- then regenerate the diagnostic if needed
- then choose `thresholds$ref_signal`
- then run `main.R`

Previous shorthand like this is incomplete:

```bash
Rscript scripts/find_ref_thresh.R
Rscript scripts/main.R
```

because it hides the required interpretation step in the middle.

Interpretation:

- in legacy mode, `thresholds$ref_signal` is a hard raw-signal filter before fold-change visualization
- in replicate-aware mode, the first value in `thresholds$ref_signal` is used to set `low_signal_flag`; analytes are still tested unless they fail separate inferential checks

### Step 5: Run `main.R`

Run:

```bash
Rscript scripts/main.R
```

What `main.R` does in legacy mode:

1. reads the manifest
2. averages duplicate membrane spots
3. maps coordinates to analytes
4. collapses sample-level data to exploratory condition-level summaries
5. filters low-signal analytes using `thresholds$ref_signal`
6. computes control-vs-treatment fold changes for configured comparisons
7. writes exploratory waterfall and barplot outputs under `main_analysis/`

What `main.R` does in replicate-aware mode:

1. reads the manifest
2. validates required columns and workbook paths
3. reads one workbook per biological sample
4. averages duplicate membrane spots within each sample
5. maps coordinates to analytes
6. builds one row per `analyte x sample_id`
7. runs within-stratum treatment-vs-control models on `log2(signal)`
8. adjusts p-values within each comparison family using the configured method, default `BH`
9. writes inferential results under `inferential_results/`

### Step 6: Run `select-analytes-analysis.R`

Run:

```bash
Rscript scripts/select-analytes-analysis.R
```

What it does:

- legacy mode: writes a focused shortlist for one configured exploratory comparison
- replicate-aware mode: reads the inferential outputs from `main.R`, selects one or more configured comparisons, and writes shortlist outputs for each comparison

This script should be run after `main.R`.

## What Each Script Does

- [scripts/install_packages.R](scripts/install_packages.R): bootstraps `pak` if needed and installs missing R packages.
- [scripts/setup/extract_analyte_table.py](scripts/setup/extract_analyte_table.py): extracts the protocol analyte-position table from the vendor PDF into an `.xlsx` workbook.
- [scripts/find_ref_thresh.R](scripts/find_ref_thresh.R): builds threshold diagnostics and suggests a low-signal reference panel.
- [scripts/main.R](scripts/main.R): runs the main analysis, dispatching to exploratory or inferential mode based on `mode` in the selected config.
- [scripts/select-analytes-analysis.R](scripts/select-analytes-analysis.R): writes a focused shortlist from the already-generated main outputs.
- [scripts/config/analysis_config.R](scripts/config/analysis_config.R): the one config file users are expected to edit.

## Output Structure

All outputs are grouped by user, then analysis:

```text
output/plots/<user>/<slug>/
```

Within that root:

- `threshold_diagnostics/`
- `main_analysis/`
- `inferential_results/`
- `select_analytes/`

`main_analysis/` is legacy exploratory output only.

`inferential_results/` exists only for replicate-aware analyses and contains:

- `run_index.tsv`
- `combined_results.xlsx`
- `comparisons/<comparison_slug>/results.tsv`

`select_analytes/` contains the focused follow-up outputs for one or more chosen comparisons.

## Statistical Meaning

Legacy mode:

- descriptive and exploratory only
- no biological replication model
- no inferential statistics
- no multiplicity correction
- `significant` in file names means "passes the fold-change threshold"

Replicate-aware mode:

- one row per analyte per biological sample
- per-analyte treatment-vs-control models within each configured stratum
- raw p-values adjusted within each comparison family
- default multiplicity correction is `BH`
- low-signal analytes are flagged separately rather than dropped by default before testing

## Methods Paragraphs

### With Biological Replicates

Proteome Profiler array signals were processed in R by averaging duplicate membrane spots within each sample, mapping spot coordinates to analytes with a protocol-derived workbook, and fitting per-analyte linear models of `log2(signal)` for treatment-versus-control comparisons within each analysis stratum. Raw p-values were adjusted within each comparison family using the Benjamini-Hochberg method, and results were summarized in tabular outputs and waterfall plots. A raw-intensity threshold derived from a low-signal reference panel was used to flag low-signal analytes.

### Without Biological Replicates

Proteome Profiler array signals were processed in R by averaging duplicate membrane spots within each membrane, mapping spot coordinates to analytes with a protocol-derived workbook, and visualizing treatment-versus-control fold changes with waterfall and barplot outputs. A raw-intensity threshold derived from a low-signal reference panel was used to exclude low-signal analytes before exploratory plotting. Because biological replicates were not available, no inferential statistical testing or multiplicity correction was performed.
